## Review fixes applied

- **FIX 1 — rearmed fails CLOSED.** `rearmed` is now a positive match against the new
  `readonly NOT_SUSPENDED_MARKER="not suspended"`; empty/unknown/paused/malformed → false.
  Proof: garbage Status `frobnicated` in a temp repo → `rearmed → exit 1`.
- **FIX 2 — bad ref → 0/1/2, never 128.** Both predicates wrap the `_status_at` pipe with
  `2>/dev/null || true`, so an unreadable ref yields empty status → false.
  Proof: `bb_done deadbeefdeadbeef → exit 1`, `rearmed deadbeefdeadbeef → exit 1` (not 128).
- **FIX 3 — deterministic session selection.** Added `STEERING_PATH` env override and
  `_resolve_steering()` (override wins, else the glob fallback); both predicates call it.
  `steering_path` now mirrors rn's rule — prefer `Status: paused` at the ref first, newest commit
  as tiebreak — with `shopt -s nullglob` (saved/restored). Proof: temp repo with an OLDER paused
  session + a NEWER not-suspended session → `steering_path → .rn/older/steering.md` (paused wins).
- **FIX 4 — CLI hardening + contract docs.** Arg-count guards reject extra positionals
  (`bb_done HEAD extra → exit 2` with usage); header now documents the tri-state exit
  (`0=true / 1=false / 2=usage/resolution error`), the `STEERING_PATH` override, and that
  `status_at` is a debug helper outside the predicate contract.
