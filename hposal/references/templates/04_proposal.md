# 提案書（アウトライン＝組み立て仕様）

お客様向けの提出物。HTMLで作り、PDFに書き出して渡す（提出物＝PDF）。**価値（お客様が得るもの）を主役**にし、技術用語は価値の後ろに小さく添えるだけにする。（書き出すスライドはお客様の言語＝日本語。）

このファイルは**提案書をどう組み立てるか（背骨・スロット順・各スロットが採るパーツ変種）**を定義する。テンプレはモノリス1枚ではなく、**1パーツ＝1スライドの部品**（`references/templates/parts/`）。`_head.html`（CSS と `<body>` 開始）＋ スロット順に選んだパーツ ＋ `_foot.html` を**この順で連結**して1つの HTML にし、`{{ }}` と `（…）` を埋め、export ゲートを通して PDF へ書き出す。CSS は `_head.html` の1か所に共有（バイト不変）。

## 背骨＝3層構造

提案書の背骨は **「要件 → オリジナルの提案（何をする・共通）→ 実現手段の提案（どう実現する・比較）」の3層**（dogfood #21）。層がレイアウトを決める：

| 層 | 内容 | レイアウト |
|---|---|---|
| **〔1層 要件〕** | 何の話で、どう読むか／いまの姿→目指す姿（共通の出発点） | **1列** |
| **〔2層 何をする・共通〕** | 画面・移行・公開・得られる価値（全案に共通のご提案） | **1列** |
| **〔3層 実現手段の比較〕** | 基盤・作り方・運用・見積を**採用候補ごとに並走** | **2列（N案ならN列）** |
| **〔締め・共通〕** | なぜ私たち・進め方・前提・取引条件・お願い・連絡先 | **1列** |
| **〔付録〕** | 用語・ページ一覧（案）・作業明細（案） | **1列** |

- **`.pg`（ページ番号）は組み立て時に採番する**。パーツには `{{頁}} / {{総ページ数}}` のプレースホルダだけが入っており、連結して順番が確定した時点で、表紙を除く各スライドへ通し番号を振る。総ページ数も組み立て時に確定。手で番号を振り直す作業は無い（パーツを足し引きしても採番し直すだけ）。
- **ページ間のクロス参照**（読み方ページの「P.◯」、前提ページの「お願いすること P.◯」など）も `{{頁}}` プレースホルダ。採番と同時に実数を入れる。

## 2層構成か3層構成か（第3層を立てるかの判断）

**第3層（実現手段の比較）を立てるのは、実現手段に施主が選ぶ本物の分岐がある時だけ**（dogfood #21）。

- **3層構成**（基盤・アプローチに施主が選ぶ分岐がある）：第3層に `compare-*` の4パーツ＋ `estimate.compare` を入れ、その層以降を2列で通す。
- **2層構成**（実現手段に本物の分岐が無い＝豆蔵 deck 型）：**第3層の `compare-*` 4パーツを丸ごと使わない**。見積は `estimate.total`（請負・スコープ固定）か `estimate.range`（準委任・実稼働精算）の1列パーツを採る。`02-read-guide` の構成リストからも比較行を外す（パーツ内コメント参照）。

## スロット順とパーツ変種

上から連結する順。「変種」のあるスロットは案件に合うものを1つ選ぶ。〔任意〕は不要なら**そのパーツを入れない**（削除・採番し直しは不要）。

### 〔1層 要件〕1列

| # | スロット | 採るパーツ | 変種・注記 |
|---|---|---|---|
| 1 | 表紙 | `01-cover` | 表紙には `.pg`/`.footmark` を置かない |
| 2 | 構成と読み方 | `02-read-guide` | 3層/2層で本編リストを変える（パーツ内コメント） |
| 3 | いまの姿→目指す姿 | `03-as-is-to-be.〔single｜multi〕` | **single**＝単一サイト移行（現行1列）／**multi**＝複数サイト統合（現行A/B/C）。#10 |

## ★ 必須／任意のパーツ — 選択のルール

変種は「削って作り直し」ではなく**パーツの選択**で決まる。

- **必須（必ず入れる）**：`01-cover`／`02-read-guide`／`03-as-is-to-be.*`（変種を1つ）／画面（`screen.*` を1〜複数）／`migration`／`value`／**`why-us`**／`process`／`premises`／`terms`／`ask`。`why-us` は特に**外さない**。
- **任意（スコープ外なら入れない）**：`post-launch-care`（公開後の見守り。準委任で固定額の見守りを積まないなら入れない＝#16）／`contact`／`appendix-cover`／付録各種。
- **3層構成のときだけ入れる**：`compare-basis`／`compare-build`／`compare-operation`／`estimate.compare`。2層構成では `estimate.total` か `estimate.range` を採る。

> 不要なパーツは**入れない**だけ。`.pg` は組み立て時に通し番号を振るので、パーツを足し引きしても番号の手振り直しは発生しない。
